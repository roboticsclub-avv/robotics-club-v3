# Robotics Club Website 3.0

## Project Goal

Build the final production-ready Robotics Club AVV website by combining:

- Version 1 functionality and business logic
- Updated Version 2 architecture and engineering practices
- Selected Version 2 visual enhancements
- Modern performance, scalability and maintainability practices

---

## Source of Truth

### Functional Source

current-v1

### Architecture Reference

reference-v2

---

## Core Principle

Version 1 remains the functional source.

Version 2 remains the architectural and visual source.

The final website should feel like Version 1 while being engineered like Version 2.

---

## Rules

- Preserve all Version 1 functionality.
- Preserve all Firebase services.
- Do not use Supabase.
- Do not use Supabase Auth.
- Do not use Supabase Database.
- Do not use SQL schemas from Version 2.
- Never modify files inside reference-v2.
- All development must happen inside current-v1.
- Maintain compatibility with:
  - Authentication
  - Recruitment workflow
  - Event workflow
  - Inventory workflow
  - Allocation workflow
  - Email notifications
